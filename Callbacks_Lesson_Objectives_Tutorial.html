First-Class Objects and Higher-Order Functions Challenge

Why are Functions called "First Class Objects" in JavaScript?

In JavaScript, functions are referred to as "First Class Objects" because they have all the characteristics of objects and can be:

  • Assigned to variables
  • Passed as arguments to other functions
  • Returned from functions
  • Stored in arrays or objects

This flexibility allows JavaScript to treat functions like any other value, making them powerful tools for writing concise, modular, and reusable code. Functions are used as "callbacks," "event handlers," and for "functional programming" techniques like map, filter, and reduce.

Predicting Output from a Code Snippet with Callbacks

Consider the following code snippet:

    function namedCallback() {{
      console.log("This is a named callback");
    }}

    setTimeout(function() {{
      console.log("This is an anonymous callback");
    }}, 1000);

    namedCallback();
    console.log("This is outside of the callback");

Predicted Output:

  • "This is a named callback" (because the named callback is called immediately)
  • "This is outside of the callback" (because this console.log is executed synchronously)
  • "This is an anonymous callback" (because the setTimeout triggers this after 1 second)

Write a function that compares two callbacks and returns the result of the greater one

    function compareCallbacks(value, cb1, cb2) {{
      const result1 = cb1(value);
      const result2 = cb2(value);

      return result1 > result2 ? result1 : result2;
    }}

    // Example usage:
    function double(x) {{
      return x * 2;
    }}

    function square(x) {{
      return x * x;
    }}

    console.log(compareCallbacks(5, double, square)); // => 25

Explanation: The function compareCallbacks accepts a value and two callback functions. It returns the result of the callback that produces the greater value.

Write a function, myMap, that mimics Array.map

    function myMap(arr, callback) {{
      const result = [];
      for (let i = 0; i < arr.length; i++) {{
        result.push(callback(arr[i], i, arr));
      }}
      return result;
    }}

    // Example usage:
    const numbers = [1, 2, 3];
    console.log(myMap(numbers, x => x * 2)); // => [2, 4, 6]

Explanation: The myMap function takes an array and a callback. It creates a new array by applying the callback function to each element in the original array, mimicking the behavior of Array.map.

Write a function, myFilter, that mimics Array.filter

    function myFilter(arr, callback) {{
      const result = [];
      for (let i = 0; i < arr.length; i++) {{
        if (callback(arr[i], i, arr)) {{
          result.push(arr[i]);
        }}
      }}
      return result;
    }}

    // Example usage:
    const numbers = [1, 2, 3, 4];
    console.log(myFilter(numbers, x => x > 2)); // => [3, 4]

Explanation: The myFilter function takes an array and a callback. It creates a new array by including only the elements for which the callback function returns true, mimicking the behavior of Array.filter.

Write a function, myEvery, that mimics Array.every

    function myEvery(arr, callback) {{
      for (let i = 0; i < arr.length; i++) {{
        if (!callback(arr[i], i, arr)) {{
          return false;
        }}
      }}
      return true;
    }}

    // Example usage:
    const numbers = [2, 4, 6];
    console.log(myEvery(numbers, x => x % 2 === 0)); // => true
    console.log(myEvery(numbers, x => x > 5)); // => false

Explanation: The myEvery function takes an array and a callback. It returns true if all elements pass the test implemented by the callback, mimicking the behavior of Array.every.
