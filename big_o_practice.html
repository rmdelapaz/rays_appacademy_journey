Big O Practice for New JavaScript Developers

Introduction to Big O Notation

Big O notation is used to describe the performance or complexity of an algorithm. It measures the worst-case time or space requirements as the input size grows.

Here are some common complexities:

  • O(1) - Constant time
  • O(log n) - Logarithmic time
  • O(n) - Linear time
  • O(n log n) - Linearithmic time
  • O(n²) - Quadratic time

Practice Problems

Problem 1: Identify the Complexity

Look at the following function and determine the time complexity:

function findFirst(arr) {
    return arr[0];
}

Question: What is the Big O complexity of this function?

Answer: O(1) - Constant time, as it only accesses the first element.

Problem 2: Nested Loop

Consider the following code:

function printAllPairs(arr) {
    for (let i = 0; i < arr.length; i++) {
        for (let j = 0; j < arr.length; j++) {
            console.log(arr[i], arr[j]);
        }
    }
}

Question: What is the time complexity of this code?

Answer: O(n²) - Quadratic time, due to the nested loops iterating through the array.

Problem 3: Logarithmic Example

Analyze this function:

function binarySearch(arr, target) {
    let left = 0;
    let right = arr.length - 1;

    while (left <= right) {
        let middle = Math.floor((left + right) / 2);
        
        if (arr[middle] === target) {
            return middle;
        } else if (arr[middle] < target) {
            left = middle + 1;
        } else {
            right = middle - 1;
        }
    }
    
    return -1;
}

Question: What is the time complexity of this function?

Answer: O(log n) - Logarithmic time, as the search space is halved each iteration.

Problem 4: Linear Time

Consider this simple function:

function sumArray(arr) {
    let sum = 0;
    for (let i = 0; i < arr.length; i++) {
        sum += arr[i];
    }
    return sum;
}

Question: What is the time complexity of this function?

Answer: O(n) - Linear time, as it loops through each element in the array once.
