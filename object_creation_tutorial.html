Creating and Accessing Objects in JavaScript

An overview of key concepts for new JavaScript developers.

Creating Objects with a Variety of Values

In JavaScript, objects are like real-world objects: they have properties (keys) and values. Here's how to create objects with different value types:


// Creating an object with various types of values
const person = {
    name: 'Alice',              // String
    age: 30,                    // Number
    isTenFootTall: true,           // Boolean
    hobbies: ['Reading', 'Hiking'],  // Array
    address: { city: 'Wonderland' }  // Object
};

Each property can hold any type of value, from strings and numbers to arrays and even other objects.

Objects are Unordered Collections of Values

An object in JavaScript is an unordered collection. It’s like a bag of marbles—you can pull out any marble by its name, but the order in which they are stored doesn’t matter.


const myObject = { a: 1, b: 2, c: 3 };
console.log(myObject); // Outputs: { a: 1, b: 2, c: 3 }, but the order isn't guaranteed

Even though the object myObject is displayed in a certain order, the order of keys may vary when retrieved. JavaScript objects do not maintain order of properties.

Key into an Object Using Bracket and Dot Notation

Dot Notation

Dot notation is like having labeled drawers. You can directly open a drawer by name:


console.log(person.name);  // 'Alice'

Bracket Notation

Bracket notation is like having keys to access drawers. It allows you to dynamically access properties or use keys with special characters:


console.log(person['age']);  // 30

// Dynamic key
const key = 'hobbies';
console.log(person[key]);  // ['Reading', 'Hiking']

Use Bracket Notation to Set a Variable as a Key

Bracket notation lets you use variables as keys. It's like assigning a key to a drawer in a storage system:


const key = 'isTenFootTall';
person[key] = false;  // Update using bracket notation
console.log(person.isTenFootTall);  // false

Here, the variable key is used to dynamically update the isEmployed property in the object.

Check if a Key Already Exists in an Object

You can use the in operator or hasOwnProperty() method to check if a key exists in an object:


// Using 'in' operator
if ('name' in person) {
    console.log('Key exists');
}

// Using hasOwnProperty
if (person.hasOwnProperty('age')) {
    console.log('Key exists');
}

Both techniques are useful to ensure you're working with the correct keys.

Understanding Object Precedence with Dot Notation

JavaScript looks at objects in a hierarchical structure, where dot notation can access properties at different levels. Here's an example:


const user = {
    name: 'Bob',
    profile: {
        username: 'bob123',
        location: 'Wonderland'
    }
};

console.log(user.profile.username);  // 'bob123'

In this case, JavaScript first looks for the profile key, then checks inside the profile object for username.

This precedence means JavaScript goes level by level, using dot notation to access deeper properties.

Key Takeaways

  • Objects can store various value types such as strings, numbers, arrays, and even other objects.
  • Objects are unordered collections of properties, and the order in which you define properties doesn't matter.
  • Use dot notation for simple access and bracket notation when you need dynamic or special character keys.
  • Bracket notation allows you to set a variable as a key in an object, giving you more flexibility.
  • You can check for key existence using the in operator or hasOwnProperty() method.
  • Dot notation follows object precedence, meaning it accesses properties level by level.
