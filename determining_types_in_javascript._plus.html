Determining Types in JavaScript

Understanding the type of value stored in a variable is essential for writing clean, predictable code. It’s like knowing the tools you have in your toolbox before starting a project. You wouldn’t use a hammer to turn a screw, and similarly, you wouldn’t call a function on a number if your function expects an array.

In this article, you’ll learn how to determine if a value is a:

  • String
  • Number
  • Function
  • Array

The typeof Operator

The typeof operator is like a label on a box. It tells you what kind of item (value) is inside. However, not all operators in JavaScript require two values (or operands), like the + for addition or = for assignment. Some, like typeof, only need one value to inspect. This type of operator is called a unary operator (think of "uni-" meaning one).

Here are some examples of using the typeof operator to check the type of a value:


    let s = 'This is a string';
    console.log(typeof s);    // 'string'

    let n = 6.28;
    console.log(typeof n);    // 'number'

    let sum = function (a, b) {
      return a + b;
    }
    console.log(typeof sum);  // 'function'

Notice that the value returned by typeof is always a string representing the type. For example, if you want to check if a value is a number, you could write:


    if (typeof n === 'number') {
      // It is a number. Let's do some math!
    } else {
      console.log('I really wanted a number. :-(');
    }

Checking for Arrays

JavaScript has a bit of a quirk when it comes to arrays. If you try to use typeof on an array, it tells you it's an "object." While arrays are technically objects, this isn't helpful when you're trying to differentiate between an array and other types of objects.


    let a = [1, 2, 3];
    console.log(typeof a);  // 'object'

To solve this, JavaScript introduced Array.isArray, a method specifically for checking if a value is an array:


    let a = [1, 2, 3];
    Array.isArray(a);  // true

    let n = 6.28;
    Array.isArray(n);  // false

    let f = function () {}
    Array.isArray(f);  // false

Practical Use in "Real" Code

Oddly enough, in many real-world applications, developers don’t often check the types of values explicitly. Most functions assume that they will receive the correct types, and parameter names often hint at what kind of value is expected. For example, a function with a parameter named sentence likely expects a string.


    function reverseTheSentence(sentence) {
      // ... code here ...
    }

Most developers will just pass in a string, assuming that passing anything else would be improper usage. However, if you expect people who aren’t familiar with your code to use it, you might want to add some checks:


    function reverseTheSentence(sentence) {
      if (typeof sentence !== 'string') {
        console.error('This function expects a string!');
        return;
      }
      // ... proceed with reversing the sentence ...
    }

This ensures that your function doesn’t break unexpectedly when someone uses it incorrectly.

Why Knowing Types Matters

Understanding and checking types can prevent bugs that are hard to track down. For example, if a function expects an array and you pass a string, the function might try to use array methods like map or forEach on the string. This will cause errors, as strings don't have these methods.

In programming, it’s like working in a kitchen. If your recipe calls for apples, and someone hands you oranges, you won’t get the result you want unless you check what you’re working with first!

Additional Methods for Determining Types

Besides typeof and Array.isArray, there are other ways to check types, especially for more complex objects:

  • instanceof: This operator checks if an object is an instance of a specific class or constructor.
  • constructor: You can use this property to check the constructor of an object.

    let today = new Date();
    console.log(today instanceof Date);  // true

    let name = 'Alice';
    console.log(name.constructor === String);  // true

Summary

In this article, we’ve covered a few ways to determine the type of a value in JavaScript:

  • The typeof operator to check if a value is a number, string, or function.
  • The Array.isArray method to check if a value is an array.
  • Other methods like instanceof and constructor for more specific type-checking needs.

Remember, determining types in your code is like knowing the ingredients in your recipe. When you know exactly what you’re working with, you can create reliable and bug-free programs. Happy coding!
