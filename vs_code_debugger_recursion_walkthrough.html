VS Code Debugger Walkthrough: Debugging Recursion

Recursion is like looking into a mirror that reflects itself endlessly. It can be confusing at first, but with the help of the VS Code debugger, you can easily follow the recursive calls and understand how they work.

Recursion Example: Factorial Function

Let's walk through an example of debugging a simple recursive function that calculates the factorial of a number.


function factorial(n) {
    if (n === 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

let result = factorial(5);
console.log(result);

Setting Breakpoints

To debug this recursive function, set a breakpoint on the line:

return n * factorial(n - 1);

Doing this will allow you to stop at each recursive call and see how the value of n changes at each step.

Starting the Debugger

Once the breakpoint is set, press F5 to start the debugger. The code will pause at the first recursive call, and you can inspect the value of n.

Stepping Through Recursive Calls

Now that you've hit the breakpoint, you can use Step Into (F11) to follow each recursive call. This is like walking through a maze and marking every turn you take, so you can trace your steps.

Each time the function calls itself, you'll notice the value of n getting smaller, until it eventually reaches 1. At that point, the function stops making recursive calls and begins to return the values back up the chain.

Watching the Stack

As you step through the recursion, you can observe the call stack. The call stack is like a stack of plates; every time a function is called, a new "plate" is added to the stack. When a function finishes, the top "plate" is removed.

In this example, each recursive call to factorial(n) adds a new entry to the stack. Once the recursion bottoms out and begins to return values, the stack shrinks as each function completes.

Inspecting Variables

At each step, you can inspect the value of n by hovering over it or checking the Variables pane. You'll see n decrease with each call, and as the recursion unwinds, you'll observe the multiplication happen as the function returns values up the stack.

Final Thoughts

Debugging recursion might seem daunting, but using the VS Code debugger makes it much clearer. You can step into each call, inspect variables, and watch how the call stack grows and shrinks. This gives you a crystal-clear view of how recursion works, like watching a loop unravel in slow motion.

Using breakpoints, stepping through, and watching the call stack are key techniques that will make debugging recursive functions much easier and more intuitive.
