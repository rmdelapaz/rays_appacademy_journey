Why Use the Square Root When Checking for Prime Numbers?

When checking if a number is prime, using the square root of the number as an upper limit for our loop significantly improves the efficiency of the algorithm. Here's why:

Key Insight: Divisors Come in Pairs

For any number N, its divisors (the numbers that can divide it without leaving a remainder) come in pairs. For example, if N = 36, some of its divisors are:

  • 1 × 36
  • 2 × 18
  • 3 × 12
  • 4 × 9
  • 6 × 6

Notice that once we pass 6 (the square root of 36), we start repeating divisors in reverse order. Therefore, if we check for divisors up to √N (in this case, √36 = 6), we are guaranteed to have checked all possible divisors without having to check larger numbers.

Why It Works

If a number N is divisible by any number greater than its square root, then it must also be divisible by some number smaller than its square root. For example, if 36 is divisible by 9, then it's also divisible by 4 (since 9 × 4 = 36).

Thus, if no number from 2 to √N divides N, then N has no divisors other than 1 and itself, meaning N is prime.

Efficiency

Checking all numbers up to N for divisibility would take O(N) time, but by stopping at √N, we reduce the complexity to O(√N). This is much faster for large numbers.

Example

Suppose you want to check if 101 is a prime number:

  • The square root of 101 is approximately 10.05.
  • You only need to check for divisors up to 10 (i.e., 2, 3, 5, 7).
  • If none of these divide 101, we conclude that 101 is prime.

Pseudocode

Here’s how the prime-checking function might look in JavaScript:

function isPrime(n) {
    if (n <= 1) return false;  // Numbers less than or equal to 1 are not prime
    if (n === 2) return true;  // 2 is prime
    if (n % 2 === 0) return false;  // Even numbers greater than 2 are not prime

    // Check for divisors up to sqrt(n)
    for (let i = 3; i <= Math.sqrt(n); i += 2) {
        if (n % i === 0) return false;  // If n is divisible by i, it's not prime
    }
    return true;  // If no divisors were found, n is prime
}

Conclusion

By using the square root of N, this function efficiently checks for primality without testing unnecessary large divisors. This approach significantly improves performance, especially for large numbers.
