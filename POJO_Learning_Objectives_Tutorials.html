Working with Objects in JavaScript

An overview of key concepts for new JavaScript developers.

Declaring an Object and Setting Keys/Values

An object in JavaScript is like a storage locker with multiple compartments. Each compartment (key) holds something valuable (value).


// Declare an object
const person = {
    name: 'Alice',
    age: 25,
    city: 'Wonderland'
};

// Access values
console.log(person.name);  // 'Alice'
console.log(person['age']); // 25

Dot Notation vs. Bracket Notation

Think of dot notation as having labeled drawers you can open directly by name. Bracket notation is like having a key to unlock a drawer, but you can use any key (including dynamic ones) as long as you have it.

When to Use Dot Notation

Use dot notation when you know the property name:


console.log(person.name); // 'Alice'

When to Use Bracket Notation

Use bracket notation when the property name is dynamic or contains characters not allowed in dot notation:


const key = 'age';
console.log(person[key]);  // 25

const specialPerson = {
    'first-name': 'Bob'
};
console.log(specialPerson['first-name']); // 'Bob'

Checking for Key Existence Using obj[key] !== undefined

To check if a key exists in an object, we can use this pattern. It’s like looking into a compartment to see if it's empty.


if (person['name'] !== undefined) {
    console.log('Name exists!');
} else {
    console.log('Name does not exist');
}

Using Object.keys() and Object.values()

Object.keys() gives you a list of all the keys in an object, like a map of all the locker compartments. Object.values() gives you all the stored items (values).


console.log(Object.keys(person));  // ['name', 'age', 'city']
console.log(Object.values(person));  // ['Alice', 25, 'Wonderland']

Iterating Through an Object with for (key in object)

This is like opening each compartment one by one and checking its content. for...in loops allow you to iterate over all keys in an object.


for (let key in person) {
    console.log(key + ': ' + person[key]);
}
// Outputs:
// name: Alice
// age: 25
// city: Wonderland

Accessing Nested Structures (Arrays and Objects)

Objects can hold other objects or arrays, like nested lockers or shelves. Accessing them requires opening each compartment layer by layer.


const company = {
    name: 'Tech Co',
    employees: [
        { name: 'Alice', role: 'Developer' },
        { name: 'Bob', role: 'Designer' }
    ]
};

console.log(company.employees[0].name);  // 'Alice'
console.log(company['employees'][1]['role']);  // 'Designer'

Using Objects as Frequency Counters

Imagine you want to count how many times each item appears in a list. An object can be used like a tally board to track occurrences.

Example: Word Frequency Counter


function countWords(words) {
    const counter = {};

    for (let word of words) {
        if (counter[word] !== undefined) {
            counter[word]++;
        } else {
            counter[word] = 1;
        }
    }

    return counter;
}

const wordList = ['apple', 'banana', 'apple', 'orange', 'banana', 'apple'];
console.log(countWords(wordList)); 
// Outputs: { apple: 3, banana: 2, orange: 1 }

Key Takeaways

  • Dot notation is used for simple, static property names, while bracket notation is useful for dynamic or special character keys.
  • You can check if a key exists using obj[key] !== undefined.
  • Object.keys() and Object.values() provide ways to list all keys and values in an object.
  • Using for...in, you can iterate over an object's keys.
  • Objects can be nested and combined with arrays for more complex data structures.
  • Objects are powerful tools for counting occurrences and managing dynamic data.
